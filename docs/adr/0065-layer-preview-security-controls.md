# Layer preview security controls

FormMuse will isolate each reviewed static Template Preview with several independent controls: minimum iframe capabilities, a preview-specific Content Security Policy, restrictive Permissions Policy, no template-initiated network behavior, and a small versioned Preview Protocol. Iframe sandboxing is not treated as a network firewall, and CSP is not treated as a replacement for capability restrictions.

The preview will not permit downloads, popups, modal browser dialogs, top-level or external-protocol navigation, storage access, real form navigation, access to FormMuse website state, or unneeded device capabilities. Preview CSP begins from denial, allows only required local scripts, styles, images, fonts, and media, and explicitly blocks connections, form actions, nested frames, objects, workers, manifests, and other unused active content. Exact directives and their static delivery mechanism must pass supported-browser and Initial Production Host testing because Next.js static hydration and animation may require narrowly scoped exceptions.

FormMuse will first test scripts without same-origin privileges and will not casually combine `allow-scripts` with `allow-same-origin` for a same-origin frame. If that model cannot load the static application, the Hanging Gifts vertical slice will record the failure and consequence before capabilities are expanded. A distinct preview origin remains an evidence-driven fallback rather than an automatic V1 infrastructure dependency. The source is owner-authored and reviewed, not community-uploaded or arbitrary, but this does not make an ineffective sandbox a security claim.

The required Preview Protocol covers readiness, Reset, and Replay only. Every message validates the expected window, origin whenever meaningful, protocol version, per-frame channel, and exact schema. It never carries visitor values, submitted content, HTML, selectors, URLs, executable code, arbitrary commands, credentials, or analytics data. Full previews use fixed viewport dimensions and internal scrolling, so no height synchronization message is required. Pause/resume may be added only if measured catalog performance proves that lazy mounting and off-screen unmounting are insufficient.

The vertical slice must publish repository-side evidence of final iframe flags, CSP, Permissions Policy, message schema, browser behavior, static-host behavior, and reasons for every permission before this configuration becomes shared preview infrastructure.
