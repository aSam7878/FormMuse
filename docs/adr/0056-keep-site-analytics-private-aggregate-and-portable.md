# Keep site analytics private, aggregate, and portable

FormMuse may collect aggregate website traffic and performance data in production so the owner can understand reach and make informed product decisions. The analytics architecture will not name or require one provider. A small site-only adapter loads from the `(site)` route group only when `FORMMUSE_DEPLOY_ENV=production` and provider configuration is present. Development, branch previews, pull-request previews, isolated Template Preview routes and iframes, registry source and output, Manual Installation, compatibility fixtures, and installed Form Templates remain analytics-free.

Any selected provider must work independently of the static hosting provider, offer a suitable free launch tier or receive explicit approval, and avoid cookies, Web Storage, fingerprinting, cross-site tracking, advertising profiles, session replay, heatmaps, and form-value collection. FormMuse may inspect aggregate pageviews, visitor estimates, referrers, broad browser/device context, coarse geography, and performance. It will not send visitor-authored fields, search text, copied code, Agent Prompts, sensitive query strings, or custom event payloads containing visitor content, and it will not create public template popularity scores in V1.

The Initial Production Host is Hostinger Premium static hosting, but neither hosting nor analytics becomes an application dependency. The portable `out/` export may move to Cloudflare Pages, Vercel, Firebase Hosting, or another static host, and the analytics provider may change independently. The production privacy notice must name the active provider and document purpose, data categories, retention, and relevant rights. CSP allows only the minimum reviewed analytics origins, and analytics failure can never impair site or registry behavior.

Cloudflare Web Analytics is the replaceable Launch Analytics Provider, selected as an operational implementation rather than added to this architecture contract. Its adapter remains site-only and may be removed or changed without affecting hosting portability, templates, registry files, documentation generation, or public URLs.
