# Defer file uploads until after V1

FormMuse V1 will launch with exactly 20 premium-quality, ready-to-paste public-facing Form Templates: ten contact, five newsletter, and five quote-request templates. Magic UI and Aceternity UI remain quality references, but matching their craft does not require FormMuse to match every component or possible form capability.

No V1 template will include an `<input type="file">`, drag-and-drop Upload Field, attachment picker, or upload-looking interaction. V1 values remain ordinary serializable form data; a form that benefits from supporting material may request a portfolio, brief, folder, or document URL instead.

Upload Fields require a distinct contract for browser `File` values, type, size and count limits, transport, progress, cancellation, storage, authorization, malware handling, privacy, retention, failure recovery, and accessible pointer and keyboard operation. FormMuse will not add dormant upload utilities, storage adapters, provider SDKs, tests, metadata, or backend infrastructure for hypothetical future use.

Upload support may be reconsidered only after demonstrated demand and would require a new decision plus dedicated templates or recipes, explicit value types, backend and security guidance, accessibility requirements, and Next.js and Vite compatibility testing. This records a possible later direction without promising a feature or timeline.
