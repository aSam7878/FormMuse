# Standardize the repository layout

FormMuse V1 will use one canonical directory structure. `app/layout.tsx` will provide a minimal shared HTML and body shell; `app/(site)/layout.tsx` will own the website navigation, footer, and site styling; and `app/(preview)/layout.tsx` will own a minimal isolated environment for static iframe preview routes. The catalog and generated Template Pages will live under the site route group, while preview routes will live under the preview route group and will not inherit the normal FormMuse shell.

The root `registry.json` will be the single authored manifest. Base UI Form Template workspaces will be colocated under `registry/base/<template-slug>/`, including their distributed component and schema, optional supporting components, repository-only examples and preview wrapper, colocated unit tests, and changelog. Shared compatibility, browser, accessibility, and visual tests will live under `tests/`; registry and catalog tooling will live under `lib/formmuse/`; generated registry JSON will live under `public/r/`; and static export output will live under `out/`.

Registry items will target `@components/formmuse/<template-slug>/` in adopter projects. Files inside that installed folder will use relative imports for other template-owned files, while shadcn Control Primitive imports will resolve through the adopter's configured local UI paths. `public/r/` and `out/` will be regenerated by the build and never edited manually, and `out/` will never be committed. Compatibility fixtures remain test fixtures rather than workspace packages, so this structure does not create a monorepo or additional maintained application.
