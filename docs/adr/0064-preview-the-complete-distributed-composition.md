# Preview the complete distributed composition

FormMuse will present each Form Template as a Complete Composition rather than reducing page-scale designs to isolated fields. A Complete Composition may be a compact newsletter section, a substantial quote-request section, or an entire scrollable contact page. The selected design determines its scale. On the Template Page, the isolated static iframe renders the complete journey from its initial entrance through the interactive form, designed result states, and every supporting section included in the template.

Preview Parity requires every section and local asset rendered inside that composition to ship in the Template Block. Only FormMuse preview chrome and the deterministic submission adapter remain repository-only. For Hanging Gifts, this permits the opening composition, hanging artwork, form, alternative contact content, and a concluding location treatment to remain when they are made generic and portable. The existing Google Maps iframe, Cedar contact details, external destinations, and Next.js-specific assets will not ship; a retained location treatment must be original local CSS/SVG artwork and the preview must not navigate externally.

The catalog will provide live Catalog Teasers rather than static-only cards. A teaser renders the opening viewport from the same static preview infrastructure, is non-interactive and excluded from the tab order, and supports an ordinary semantic card link and description in the parent page. The Template Page remains the place for complete scrolling, field entry, validation, deterministic submission, result states, device controls, Reset, and Replay Animation.

Catalog performance will not depend on eagerly mounting all 20 complete previews. Card content and reserved teaser space render immediately; live teasers activate only near the viewport and off-screen animation pauses or unmounts so previously viewed cards do not continue consuming CPU. Reduced-motion behavior applies in both contexts. The Hanging Gifts vertical slice must measure loading, main-thread work, memory, scrolling, and animation before FormMuse fixes activation thresholds or numeric performance budgets.

The preview itself makes no template-initiated or external request. Same-origin static files required to load the generated preview document are measured as preview transport; publication tests establish an idle post-load baseline and then reject unexpected requests during field interaction, submission simulation, resetting, replaying, and scrolling. No runtime sandbox, backend, external map, analytics beacon, or provider service is introduced.
