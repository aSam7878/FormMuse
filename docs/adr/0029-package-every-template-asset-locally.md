# Package every Template Asset locally

Form Templates will prefer, in order, CSS artwork, inline or template-owned JSX/TSX SVG, Lucide icons, and optimized local raster images only when genuinely concept-essential. Templates and Template Previews will not fetch remote images, illustrations, icons, videos, textures, or other media at runtime. They will not use `next/image`, external CDNs, large base64 data URLs outside the narrow registry transport exception below, or SVG-loader conventions that differ between Next.js and Vite.

Every Template Asset rendered inside the isolated preview iframe must ship in the corresponding Template Block; FormMuse preview chrome outside the iframe is not part of that template payload. Raster assets must be optimized, explicitly installed, original or legally Redistribution-safe, and verified in both Next.js and Vite. Repository-only documentation will record asset provenance and licence terms.

Meaningful images require useful `alt` text. Purely decorative SVGs, icons, and images must use appropriate empty alternative text or `aria-hidden="true"` treatment and must not create distracting accessibility-tree noise. Zero-network preview tests will verify that every in-template asset resolves locally.

When a Form Template genuinely requires raster files, its Template Block will install them as `registry:file` entries under `~/public/formmuse/<canonical-slug>/`. That template—and only a template with such assets—will expose `assetBaseUrl?: string`, defaulting to `/formmuse/<canonical-slug>`. It will normalize trailing slashes before constructing URLs and use standard `<img>` elements with explicit intrinsic dimensions and appropriate accessibility attributes. FormMuse previews always use the local default; adopters may explicitly override the value with a deployment subpath or their own absolute CDN URL.

The only Base64 exception is a standalone, template-owned SVG `registry:file` used when the pinned shadcn builder cannot preserve an approved local raster's binary bytes. The SVG may contain exactly that raster as a Base64 data URI and must remain a local public asset addressed through `assetBaseUrl`; Base64 must never be embedded in TypeScript, TSX, JavaScript, JSX, CSS, or HTML. Registry generation must be deterministic, decode to the approved raster's exact byte length and SHA-256, enforce the reviewed size cost and locality, and reject external requests. The compatibility pipeline must render the asset in the supported Next.js and Vite fixtures and automated browser engines and prove that it works under the preview image CSP without a `data:` allowance. This exception changes only the byte-safe registry transport, not the asset's provenance, licence, accessibility treatment, locality, or later branded-browser and real-device publication gates.
